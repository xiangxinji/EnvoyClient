## 3. 数据库设计

### 3.1 数据库规范

- 所有表使用 `utf8mb4` 字符集
- 主键使用 `BIGINT UNSIGNED AUTO_INCREMENT`
- 必须包含 `created_at` 和 `updated_at` 时间字段
- 逻辑删除使用 `deleted_at` 字段

### 3.2 核心表结构（示例）

**users 用户表**

```sql
CREATE TABLE users (
    id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    username    VARCHAR(64)  NOT NULL UNIQUE COMMENT '用户名',
    email       VARCHAR(128) NOT NULL UNIQUE COMMENT '邮箱',
    password    VARCHAR(256) NOT NULL COMMENT '加密密码',
    status      TINYINT      NOT NULL DEFAULT 1 COMMENT '状态: 1-正常 0-禁用',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at  DATETIME     DEFAULT NULL,
    INDEX idx_email (email),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

**orders 订单表**

```sql
CREATE TABLE orders (
    id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    order_no      VARCHAR(32)  NOT NULL UNIQUE COMMENT '订单号',
    user_id       BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
    total_amount  DECIMAL(12,2) NOT NULL COMMENT '总金额',
    status        TINYINT      NOT NULL DEFAULT 0 COMMENT '状态: 0-待支付 1-已支付 2-已发货 3-已完成 4-已取消',
    paid_at       DATETIME     DEFAULT NULL COMMENT '支付时间',
    created_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at    DATETIME     DEFAULT NULL,
    INDEX idx_user_id (user_id),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表';
```

## 7. 安全方案

### 7.1 网络安全

- 所有外部请求通过 WAF 过滤
- API 接口全量 HTTPS
- 内部服务间通信使用 mTLS

### 7.2 数据安全

- 用户密码使用 bcrypt 加密存储
- 敏感数据（手机号、身份证）使用 AES-256 加密
- 数据库访问使用最小权限原则
- 定期进行数据备份和恢复演练

### 7.3 认证与授权

- JWT Token 认证，有效期 2 小时
- Refresh Token 有效期 7 天
- RBAC 权限模型，支持细粒度权限控制
