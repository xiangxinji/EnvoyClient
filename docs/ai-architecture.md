## 二、AI 功能分层架构图

AI 系统分为 **四个独立层**，每层有独立的 composable、API 端点和错误状态，互不干扰：

```
┌─────────────────────────────────────────────────────────────────┐
│                      AI 功能四层架构                              │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Layer 1: 聊天建议 (useAIChat)                           │   │
│  │  ─────────────────────────────                           │   │
│  │  触发: 用户点击闪电按钮                                    │   │
│  │  通信: SSE 流式 (POST /api/ai/chat/stream)                │   │
│  │  输出: suggestion ref → 输入框建议文本                     │   │
│  │  错误: chatAiError (独立状态)                              │   │
│  │  使用方: ChatPanel                                        │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Layer 2: 自动回复 (useAutoReply)                         │   │
│  │  ─────────────────────────────                           │   │
│  │  触发: 收到消息 + ai_auto_reply 开启                       │   │
│  │  通信: 普通 POST (POST /api/ai/auto-reply/generate)       │   │
│  │  输出: 自动调用 sendChat({ source: "ai-auto" })           │   │
│  │  防抖: 每 peer 5 秒                                       │   │
│  │  使用方: useTeamClient                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Layer 3: 任务 AI (useAITask)                             │   │
│  │  ─────────────────────────────                           │   │
│  │  功能: plan / dispatch / analyze / review                 │   │
│  │  通信: 普通 POST (4 个端点)                                │   │
│  │  错误: taskAiError (独立状态)                              │   │
│  │  使用方: TaskDispatchPanel, useTaskExecution              │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Layer 4: Agent 执行 (useTaskExecution + useAgent)        │   │
│  │  ─────────────────────────────                           │   │
│  │  触发: WebSocket "doing" 事件 + task_execution_mode=auto  │   │
│  │  通信: ReAct 循环 (POST /api/ai/agent/reason × N 步)      │   │
│  │  工具: shell, file_read, file_write, cloud, done 等       │   │
│  │  限制: 最多 20 步, 工具超时 60s, 推理超时 120s             │   │
│  │  使用方: useTeamClient → useTaskExecution                 │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

## 八、WebSocket 连接生命周期

```
┌────────┐                                                  ┌──────────┐
│ Client │                                                  │  Manager  │
│        │                                                  │  WS Server│
└───┬────┘                                                  └────┬─────┘
    │                                                            │
    │  1. new WebSocket(ws://{host}:{team.port})                │
    │ ────────────────────────────────────────────────────────► │
    │                                                            │
    │  2. onopen                                                 │
    │ ◄────────────────────────────────────────────────────────  │
    │      │                                                     │
    │      ▼                                                     │
    │  3. 状态: connected                                        │
    │     发送 team:join                                         │
    │ ────────────────────────────────────────────────────────► │
    │                                                            │
    │  ══════════ 正常通信阶段 ══════════                         │
    │                                                            │
    │  4. 接收 message 事件 (聊天消息)                             │
    │ ◄────────────────────────────────────────────────────────  │
    │                                                            │
    │  5. 接收 task 事件 (任务通知)                                │
    │ ◄────────────────────────────────────────────────────────  │
    │                                                            │
    │  6. 接收 doing 事件 (执行指令)                               │
    │ ◄────────────────────────────────────────────────────────  │
    │                                                            │
    │                                                            │
    │  ══════════ 异常断开 ══════════                             │
    │                                                            │
    │  7. onclose (非 4001)                                      │
    │ ◄────────────────────────────────────────────────────────  │
    │      │                                                     │
    │      ▼                                                     │
    │  8. 快速重连阶段 (指数退避)                                   │
    │     attempt 1: 3s    ──► 连接失败                           │
    │     attempt 2: 6s    ──► 连接失败                           │
    │     attempt 3: 9s    ──► 连接失败                           │
    │     attempt 4: 12s   ──► 连接失败                           │
    │     attempt 5: 15s   ──► 连接失败                           │
    │     ...                                                    │
    │     attempt 10: 15s  ──► 连接失败                           │
    │      │                                                     │
    │      ▼                                                     │
    │  9. 慢速重连阶段 (30s 无限重试)                               │
    │     emit("reconnect_failed")                               │
    │     每 30s 重试一次 ──► 直到成功                              │
    │                                                            │
    │                                                            │
    │  ══════════ 重复登录被拒 ══════════                          │
    │                                                            │
    │  10. onclose (code 4001 = DUPLICATE_LOGIN)                 │
    │ ◄────────────────────────────────────────────────────────  │
    │      │                                                     │
    │      ▼                                                     │
    │  11. emit("rejected")                                      │
    │      显示 "账号在其他设备登录" 提示                            │
    │      不触发重连                                              │
    └────────────────────────────────────────────────────────────┘
```
