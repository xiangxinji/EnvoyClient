## 3. 核心组件设计

### 3.1 任务分析模块

**功能描述**：
- 解析用户输入的自然语言任务
- 提取任务目标、约束条件和预期结果
- 将复杂任务拆解为可执行的子步骤

**工作逻辑**：
1. 接收用户任务指令
2. 识别任务类型（文件操作、信息查询、代码执行等）
3. 分解任务步骤，生成执行计划
4. 确认关键约束条件

### 3.2 执行决策模块

**功能描述**：
- 根据任务分析结果，决定下一步操作
- 选择合适的工具和参数
- 处理执行过程中的异常情况

**决策流程**：
```
分析任务 → 判断当前状态 → 选择工具 → 执行 → 评估结果
    ↑                                            │
    └──────────────── 循环迭代 ──────────────────┘
```

### 3.3 工具调用模块

**功能描述**：
- 封装工具调用接口
- 传递参数并获取执行结果
- 处理工具返回的错误信息

### 3.4 结果评估模块

**功能描述**：
- 评估工具执行结果是否符合预期
- 判断任务是否完成
- 生成执行报告

## 4. 工作流程

### 4.1 标准执行流程

```
步骤 1: 接收任务
    │
    ▼
步骤 2: 分析任务需求
    │
    ▼
步骤 3: 制定执行计划
    │
    ▼
步骤 4: 调用工具执行
    │
    ▼
步骤 5: 观察执行结果
    │
    ├── 成功 → 判断是否完成
    │               │
    │         ├── 是 → 步骤 6: 提交结果
    │         └── 否 → 返回步骤 4
    │
    └── 失败 → 分析原因 → 尝试替代方案 → 返回步骤 4
```

### 4.2 异常处理流程

| 异常类型 | 处理策略 |
|----------|----------|
| 工具执行失败 | 分析错误原因，尝试替代方案 |
| 参数错误 | 检查参数格式，重新调用 |
| 权限不足 | 检查操作权限，调整执行策略 |
| 资源不存在 | 确认资源路径，创建或查询可用资源 |

## 5. 工具接口规范

### 5.1 可用工具列表

| 工具名称 | 功能 | 参数 |
|----------|------|------|
| `shell` | 执行 Shell 命令 | `command: string` |
| `file_read` | 读取文件内容 | `path: string` |
| `file_write` | 写入文件内容 | `path: string`, `content: string` |
| `done` | 声明任务完成 | `result: string` |
| `upload_resource` | 上传资源文件 | `path: string` |
| `query_resources` | 查询任务资源 | `taskId: string` |
| `read_resource` | 读取资源文件 | `taskId: string`, `file: string` |

### 5.2 工具调用规范

```json
{
  "tool_call": {
    "tool_name": "shell",
    "parameters": {
      "command": "ls -la"
    }
  }
}
```

### 5.3 返回值规范

所有工具返回 JSON 格式结果，包含：
- `exit_code`: 执行状态码（0 成功，非 0 失败）
- `stdout`: 标准输出内容
- `stderr`: 错误输出内容

## 6. 安全与约束

### 6.1 操作安全规范

| 规则 | 说明 |
|------|------|
| 禁止危险命令 | 避免 `rm -rf /`、`format` 等破坏性操作 |
| 路径安全 | 操作限定在指定工作目录范围内 |
| 参数校验 | 调用工具前校验参数合法性 |

### 6.2 执行约束

- 每一步操作前必须进行思考分析
- 工具执行失败时需分析原因并尝试替代方案
- 任务完成后必须调用 `done` 工具提交结果
- 合理使用 Shell 命令，避免不必要的资源消耗

## 7. 最佳实践

### 7.1 任务执行建议

1. **明确目标**：清晰理解任务需求后再执行
2. **分步执行**：复杂任务拆解为多个简单步骤
3. **及时验证**：每一步执行后检查结果
4. **异常处理**：遇到错误时分析原因，寻找替代方案
5. **完整交付**：任务完成后整理结果并提交
